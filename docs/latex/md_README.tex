GUB Game Engine is a powerful and versatile engine designed for the development of game and graphic applications across multiple platforms, including Android, Linux, and Windows. Whether you\textquotesingle{}re building a 2D or 3D game, GUB provides the tools and flexibility you need to bring your projects to life.\hypertarget{md_README_autotoc_md1}{}\doxysection{\texorpdfstring{Features}{Features}}\label{md_README_autotoc_md1}

\begin{DoxyItemize}
\item {\bfseries{Cross-\/\+Platform Support}}\+: Develop once, deploy on Android, Linux, and Windows.
\item {\bfseries{Customizable Components}}\+: Design custom UI components, inspired by React, for dynamic and interactive game interfaces.
\item {\bfseries{Software Rendering}}\+: Pure software rendering engine, with no hardware acceleration required, suitable for projects where control over every pixel is essential.
\item {\bfseries{Z-\/\+Buffer Implementation}}\+: Efficient depth management for 3D rendering.
\item {\bfseries{Event Handling System}}\+:
\begin{DoxyItemize}
\item Utilizes a pattern involving hash tables of lists of nodes.
\item Each node contains a listener that handles events, with functionalities for adding listeners similar to how Java\+Script manages events.
\item Includes support for keyboard, mouse, and joystick event handling, providing comprehensive input management for diverse gaming experiences.
\end{DoxyItemize}
\item {\bfseries{Extensible Architecture with Facade Pattern}}\+: The engine can be easily extended with other graphic libraries via the facade pattern. This is how it supports both SDL 1 and SDL 2, providing flexibility in choosing or integrating other libraries.
\end{DoxyItemize}\hypertarget{md_README_autotoc_md2}{}\doxysection{\texorpdfstring{Getting Started}{Getting Started}}\label{md_README_autotoc_md2}

GUB Game Engine\textquotesingle{}s architecture allows for easy extension with other graphic libraries using the facade pattern. This design pattern abstracts the underlying graphic library, enabling seamless support for SDL 1, SDL 2, and potential future libraries. To integrate a new library, implement the necessary facade interfaces in the engine and adjust the build scripts to include the new library.\hypertarget{md_README_autotoc_md10}{}\doxysubsection{\texorpdfstring{Sample Projects}{Sample Projects}}\label{md_README_autotoc_md10}
